# “编辑部档案馆”前端重设计规格

**日期**：2026-06-20

**状态**：已通过用户确认

**适用范围**：`prototype/` 下 6 个静态原型页面

## 1. 目标

将现有“智识花园”原型从常见的深色渐变玻璃风格重设计为具有个人辨识度的“编辑部档案馆”。新设计同时服务两类内容：面向访客的公开博客，以及仅作者可访问的私密笔记。

本轮直接修改现有静态原型，用它作为后续 Nuxt 3 / Vue 3 实现的视觉与交互基线，不在本轮引入框架迁移、后端接口或真实鉴权。

## 2. 页面范围

| 页面 | 文件 | 核心职责 |
| --- | --- | --- |
| 首页 | `prototype/index.html` | 刊物封面、最新内容、作者档案 |
| 博客列表 | `prototype/blog.html` | 公开文章检索、筛选和浏览 |
| 博客详情 | `prototype/blog-post.html` | 长文阅读、目录与相邻文章导航 |
| 笔记列表 | `prototype/notes.html` | 私密档案检索、筛选和最近更新 |
| 笔记详情 | `prototype/note-post.html` | 私密内容阅读、编辑与转公开入口 |
| 登录页 | `prototype/login.html` | 单用户身份验证演示 |

共享资源继续使用：

- `prototype/assets/styles.css`
- `prototype/assets/app.js`

## 3. 设计原则

1. **内容优先**：先用版式、留白、字体和分隔线建立层级，再使用底色、边框和阴影。
2. **公开与私密明确分区**：博客采用“刊物”语义；笔记采用“档案”语义，权限状态在页面首屏可见。
3. **减少卡片泛滥**：普通文章列表使用共享表面上的目录行，不把每条内容都放进独立悬浮卡片。
4. **阅读舒适**：正文控制在约 60–65 个中文字符的视觉行宽，标题、导语、元信息和正文形成稳定节奏。
5. **移动端不是缩小版桌面**：导航、筛选、目录和详情操作使用适合触控的独立交互。
6. **原型可迁移**：共享视觉模式应能自然映射到后续 Vue 组件和设计令牌。

## 4. 视觉系统

### 4.1 方向

整体采用“编辑部档案馆”语言：暖白纸张、墨黑正文、朱红索引色、细线分栏、刊物编号和档案标签。移除现有大面积极光渐变、过量透明玻璃面板和同质化卡片。

### 4.2 色彩

浅色主题：

- 页面纸张：暖白/浅米色
- 主要文字：墨黑
- 次要文字：暖灰
- 强调色：朱红，用于当前状态、关键动作和编号
- 辅助色：深松绿，仅用于成功、公开/已发布等状态
- 分隔线：低对比暖灰

深色主题：

- 页面纸张：深棕黑，而不是纯黑
- 主要文字：柔和米白
- 次要文字：灰褐色
- 强调色继续使用朱红，但降低亮度刺激
- 分隔线与表面层级重新定义，不做浅色主题的机械反色

所有颜色通过 CSS 自定义属性管理，并达到正文与交互控件的基本对比度要求。

### 4.3 字体

- 中文展示标题使用具有宋体气质的字体，承担刊名、页面标题和文章标题。
- 正文与界面文字使用高可读黑体。
- 日期、编号和代码使用等宽字形，但不再引入第三套独立字体文件；优先使用正文或系统字体栈的等宽回退。
- 一个页面最多使用两套主字体体系，避免字体角色过多。

### 4.4 版式和表面

- 以编辑网格、留白和细线分隔建立页面结构。
- 首页允许不对称编排，列表和详情页保持稳定阅读网格。
- 卡片仅保留给确实独立的对象或需要强调的内容，普通列表改为文章目录行。
- 背景可使用非常轻微的纸张纹理，但不得影响文字清晰度或滚动性能。
- 动效围绕页面进入、列表状态变化、抽屉和目录切换展开，避免装饰性漂浮动画。

## 5. 全局导航与响应式结构

### 5.1 桌面端

- 顶部导航改为刊头式布局，品牌、公开导航、私密入口和主题切换具有清晰分区。
- 当前页面通过编号、下划线和朱红状态共同标识，不依赖大面积胶囊背景。
- 导航在滚动后保持可用，但高度和视觉重量降低，避免遮挡正文。

### 5.2 移动端

- 头部保留品牌、主题切换和菜单按钮。
- 菜单使用可关闭的抽屉；打开后锁定背景滚动，焦点进入抽屉，关闭后返回触发按钮。
- 主要触控目标不小于 `44px × 44px`。
- 不使用横向溢出的固定桌面布局；仅标签筛选条允许受控横向滚动。
- 页面底部操作必须避开安全区，并为系统手势预留空间。

## 6. 页面设计

### 6.1 首页

首页由“本期封面”“最新文章目录”“作者档案”三部分组成。

- 首篇文章作为主视觉内容，展示期号、发布日期、标题、摘要与阅读入口。
- 其余文章使用带编号的目录行，展示类别、时间和阅读时长，减少三张同构卡片。
- 作者介绍采用档案式信息布局，关注领域和联系方式保持简洁。
- 公开浏览和私密入口均可见，但私密入口采用权限语义，避免与公开主行动竞争。

### 6.2 博客列表

- 页面顶部提供搜索框、标签筛选条和结果数量。
- 搜索词与多个选中标签同时生效，不再互相覆盖显示状态。
- 标签在移动端横向滚动；当前标签具备明确的视觉与 ARIA 选中状态。
- 文章以目录行显示标题、摘要、标签、日期和阅读时长。
- 无匹配内容时展示空状态和单一的“清除条件”操作。

### 6.3 博客详情

- 正文保持舒适行宽，标题、导语、元信息、正文和注释层级明确。
- 桌面端目录吸附在右侧，当前章节随滚动自动更新。
- 移动端目录收进底部抽屉，并显示阅读进度；目录入口不持续占用正文横向空间。
- 上一篇/下一篇使用轻量目录行，不使用两张厚重卡片。

### 6.4 笔记列表

- 首屏明确显示“私密档案”和当前访问身份，避免仅靠锁图标表达权限。
- 搜索和标签筛选行为与博客一致，复用同一交互模型。
- 桌面端允许主列表与最近更新摘要并列；移动端将二者合并为单列时间顺序，避免窄屏双栏。
- 草稿、灵感、日记等状态通过文本与颜色共同表达。

### 6.5 笔记详情

- 阅读结构延续博客详情，但使用档案编号、私密状态和更新时间区分内容属性。
- 保留“编辑”和“转为公开博客”操作。
- 桌面端操作位于文章尾部；移动端使用安全区内的吸底操作栏。
- “转为公开博客”改变内容身份，原型中必须先出现确认步骤，避免误触。

### 6.6 登录页

- 桌面端采用刊物封面信息与登录表单左右分栏，不再把表单作为孤立玻璃卡片悬浮在背景上。
- 移动端隐藏非必要说明，将焦点放在用户名、密码和提交操作。
- 增加密码显示/隐藏、提交中、失败提示和字段级错误状态。
- 保留“任意账号密码可进入”的原型说明，但与真实产品文案视觉隔离。

## 7. 交互与状态

### 7.1 主题

- 保留 `localStorage` 主题记忆。
- 所有页面的主题切换按钮状态同步更新。
- 若没有本地偏好，首次访问遵循系统主题。
- 主题切换更新 `theme-color`，并提供清晰的按钮名称。

### 7.2 导航与抽屉

- 当前导航项根据 `body[data-page]` 设置。
- 移动抽屉支持按钮、遮罩和 `Escape` 关闭。
- 增加焦点管理、`aria-expanded`、`aria-hidden` 和背景滚动锁定。

### 7.3 搜索与标签筛选

- 使用单一状态计算可见文章：`文本匹配 AND 标签匹配`。
- 多标签采用“任一已选标签匹配”语义。
- 搜索词和标签写入 URL 查询参数，刷新后可以恢复。
- 筛选结果变化后更新结果数量与空状态。
- 清除操作同时清除搜索词、标签和 URL 状态。

### 7.4 文章目录

- 根据标题元素生成或绑定目录。
- 使用滚动观察更新当前章节。
- 点击目录平滑定位；启用“减少动态效果”时关闭平滑滚动。
- 无可用标题时隐藏目录入口，不保留空容器。

### 7.5 登录演示

- 表单在前端模拟加载和错误状态，不调用真实接口。
- 空字段使用浏览器约束与可见错误提示。
- 密码显示按钮不会改变输入值或丢失焦点。
- 成功后跳转首页，保持原型可点击闭环。

## 8. 异常与边界状态

- 无搜索结果：显示当前条件摘要和清除操作。
- 标签过多：移动端横向滚动，桌面端允许换行；清除按钮始终可达。
- 登录失败：错误与输入关联，不只使用颜色表达。
- 提交中：按钮禁用并保持稳定宽度，防止重复提交。
- 无目录：隐藏目录及触发按钮。
- 长标题：允许合理换行，不截断文章详情主标题。
- JavaScript 不可用：页面导航和文章链接仍可工作，搜索与筛选作为增强功能退化。
- `prefers-reduced-motion: reduce`：关闭页面进入、平滑滚动和抽屉位移动画。

## 9. 代码边界

本轮仅修改：

- `prototype/*.html`
- `prototype/assets/styles.css`
- `prototype/assets/app.js`

不包含：

- Nuxt 3 脚手架或 Vue 组件实现
- 后端接口、SQLite、鉴权、OSS 或部署配置
- 新增内容管理、编辑器或后台页面
- 真实用户数据和网络请求

共享 CSS 和 JavaScript 需要按职责组织，避免继续增加页面内联样式。页面专属差异优先通过语义类或 `data-*` 属性表达。

## 10. 验收标准

### 10.1 视口

至少检查以下宽度：

- `375px`：手机
- `768px`：平板/窄桌面
- `1440px`：桌面

六个页面在三档视口下均不得出现非预期横向滚动、遮挡、裁切或不可达操作。

### 10.2 功能

- 六页之间导航闭环有效。
- 主题切换、持久化和系统偏好有效。
- 移动抽屉可通过按钮、遮罩和键盘关闭。
- 博客与笔记的搜索、组合标签筛选、URL 恢复、结果计数和空状态有效。
- 文章目录跟随章节，桌面与移动入口均有效。
- 登录页密码显示、加载、错误和成功跳转状态有效。
- 笔记转公开操作具备确认步骤。

### 10.3 可访问性与质量

- 所有交互控件可通过键盘访问，并有可见焦点。
- 主要触控目标不小于 44px。
- 标题结构、地标和表单标签语义合理。
- 图标使用现有图标资源或成熟图标库，不使用文本符号、Emoji 或 CSS 图形替代。
- 尊重减少动态效果设置。
- HTML、CSS 和 JavaScript 通过项目可用的静态检查；控制台无运行时错误。

## 11. 实施顺序

1. 重建设计变量、基础排版和主题。
2. 重构刊头导航、移动抽屉和通用页脚。
3. 实现首页封面与目录结构。
4. 实现博客/笔记共享列表和筛选状态。
5. 实现博客/笔记共享详情阅读结构与响应式目录。
6. 实现登录页状态。
7. 完善移动端、无障碍与减少动态效果。
8. 在 3 档视口逐页验证并修复。
